# Code Deduplication Design

**Date:** 2026-04-06
**Goal:** Reduce SonarCloud duplicated lines density from 3.37% to as close to 0% as possible.
**Approach:** Biggest Impact First (Approach B) — 6 sequential PRs, each targeting a duplication category.
**Estimated total reduction:** ~2,065 duplicated lines.

---

## PR 1: Test Helper Centralization (~950 lines)

**Problem:** `renderWithChakra` is copy-pasted in 34 test files. Common `vi.mock()` patterns for `next/navigation` (28 files), `@/lib/supabase/client` (28 files), `@/components/ui/toaster` (23 files), and others are repeated with minor variations.

**Changes:**

1. Delete all 34 local `renderWithChakra` definitions; replace with import from `@/__tests__/helpers/mocks`.

2. Add shared mock factories to `src/__tests__/helpers/mocks.ts`:
   - `createMockNavigation(overrides?)` — covers `useRouter`, `useSearchParams`, `redirect`
   - `createMockSupabaseClient(overrides?)` — covers auth-only, auth+from, from-only variants
   - `createMockToaster()` — standardized toaster mock
   - `createMockNextLink()` — the `<a>` wrapper mock (15 files)
   - `createMockColorMode()` — `ColorModeButton: () => null` mock (15 files)
   - `createMockField()` — the `Field` wrapper mock (9 files)
   - `createMockPasswordInput()` — password input mock (6 files)

3. Each test file keeps its own `vi.mock()` calls (required by Vitest hoisting) but passes shared factories:
   ```ts
   vi.mock("next/navigation", () => createMockNavigation({ push: mockPush }));
   ```

4. `vi.hoisted()` blocks stay per-file (test-specific mock references).

**Files affected:** 34+ test files in `src/__tests__/`, `src/__tests__/helpers/mocks.ts`

## PR 2: Shared Layout Components (~250 lines)

**Problem:** Admin and Dashboard have near-identical Sidebar (~120 lines dup), Header (user loading ~70 lines dup), and Shell (~13 lines dup) components.

**Changes:**

1. **`src/components/shared/BaseSidebar.tsx`** — Unified sidebar:
   - Props: `navItems`, `logoIcon`, `logoText`, `logoSubtext`, `accountSection` (ReactNode slot), `onSignOut`
   - Handles mobile nav (fixed bottom) and desktop nav (fixed left)
   - `AdminSidebar` and `DashboardSidebar` become thin wrappers (~15 lines each)

2. **`src/lib/hooks/useUserProfile.ts`** — Custom hook:
   - Encapsulates: `getUser()` -> student lookup -> staff fallback -> avatar signed URL
   - Returns `{ userName, avatarUrl, loading }`
   - Options: `{ includeAvatar?: boolean }`
   - Replaces duplicated logic in `AdminHeader`, `DashboardHeader`, `DashboardSidebar`

3. **`src/components/shared/LayoutShell.tsx`** — Unified shell:
   - Props: `sidebar` (ReactNode), `children`, `providers?` (ReactNode wrapper)
   - Handles `Box > Flex > sidebar + content` with responsive margin/padding
   - `AdminShell` and `DashboardShell` become thin wrappers

**Files affected:** `AdminSidebar.tsx`, `DashboardSidebar.tsx`, `AdminHeader.tsx`, `DashboardHeader.tsx`, `AdminShell.tsx`, `DashboardShell.tsx`

## PR 3: Shared UI Patterns (~350 lines)

**Problem:** Dialog boilerplate (~35 lines) repeated across 3 confirmation dialogs. Skeleton card patterns (~150 lines) repeated across 4 files. Checklist structure (~25 lines) duplicated between GenEd and Major.

**Changes:**

1. **`src/components/shared/ConfirmationDialog.tsx`**:
   - Props: `open`, `onOpenChange`, `onConfirm`, `title`, `children`, `confirmText`, `confirmColor`, `loading`
   - Encapsulates: `Dialog.Root` -> `Portal` -> `Backdrop` -> `Positioner` -> `Content` -> `Header/Body/Footer` -> `CloseTrigger`
   - `DeletePlanDialog`, `RemoveSemesterDialog`, `AddSemesterDialog` shrink to ~15-20 lines each

2. **`src/components/shared/SkeletonParts.tsx`**:
   - `SkeletonCard` — Card.Root with standard border/radius + header/body slots
   - `SkeletonRow` — HStack with icon placeholder + text skeleton
   - `SkeletonProgressBar` — HStack + full-width skeleton bar
   - Each skeleton page keeps its layout but uses shared building blocks

3. **`src/components/shared/RequirementCard.tsx`**:
   - Props: `title`, `badge` (ReactNode), `items: Array<{ id, label, checked }>`, `onToggleItem`
   - `GenEdChecklist` and `MajorChecklist` become thin wrappers

**Files affected:** `DeletePlanDialog.tsx`, `RemoveSemesterDialog.tsx`, `AddSemesterDialog.tsx`, `CourseSearchDialog.tsx`, `DashboardSkeleton.tsx`, `PlannerSkeleton.tsx`, `SettingsSkeleton.tsx`, `RequirementsSkeleton.tsx`, `GenEdChecklist.tsx`, `MajorChecklist.tsx`

## PR 5: Query Layer Cleanup (~260 lines)

**Problem:** Redundant function aliases for `viewItemToCourse` (4 files), duplicated block mapping (3 files), duplicated planned course mapping, and repeated auth checks in API routes.

**Changes:**

1. Remove local aliases (`toCourseFromBlockItem`, `toCourseRowFromView`, `toCourseRow`) in `planner.ts`, `onboarding.ts`, `classHistory.ts`; use `viewItemToCourse` directly.

2. Add `mapViewBlockToCourseBlock()` to `helpers.ts`; use in `planner.ts`, `onboarding.ts`, `classHistory.ts`.

3. Have `planner.ts` import `mapPlannedCourses` from `shared-plans.ts` instead of duplicating it.

4. Add `requireAuthUser()` to `src/lib/auth-helpers.ts`:
   - Does `supabase.auth.getUser()` + 401 response pattern
   - Returns `{ user, supabase }` or `NextResponse`
   - Used by all API routes under `src/app/api/`

**Files affected:** `planner.ts`, `onboarding.ts`, `classHistory.ts`, `helpers.ts`, `shared-plans.ts`, `auth-helpers.ts`, API route files

## PR 6: Miscellaneous Cleanup (~75 lines)

**Problem:** Inline subject color function with divergent mappings, repeated email domain validation, identical layout wrappers, scattered pagination constants.

**Changes:**

1. **Subject colors:** Delete inline `getSubjectColor()` in `ProgramDetailClient.tsx`; import from `@/lib/subject-colors`. Fixes 6 color inconsistencies.

2. **Email validation:** Create `src/lib/email-validation.ts` with `validateEmailDomain(role, email)`. Replace inline checks in `RoleSignInForm.tsx`, `AdvisorSignupClient.tsx`, `signup/page.tsx`.

4. **Pagination constants:** Create `src/lib/constants.ts` with `ADMIN_PAGE_SIZE`, `DASHBOARD_PAGE_SIZE`, `API_PAGE_SIZE`. Import in respective files.

**Files affected:** `ProgramDetailClient.tsx`, `RoleSignInForm.tsx`, `AdvisorSignupClient.tsx`, `signup/page.tsx`, `shared/layout.tsx`, `CoursesAdminClient.tsx`, `CoursesClient.tsx`, `queries/courses.ts`
